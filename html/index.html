ZC706 Diagnostic Tool Demonstration

Introduction

ZC706_SPI_Display.c - Display data retrieved from ZC706 via SPI port.
The data is displayed using TI EK-TM4C123GXL launch pad and
Kentec 320x240 (EB-LM4F120-L35) display.
Code uses the Stellaris Graphics Library and Tiva Ware (TM) Peripheral Driver Library

SPI Interface


This code configures the serial port SSI1 as SPI Slave on an
TM4C123G launchpad (EK-TM4C123GXL) evaluation board.
An RX timeout interrupt is configured for SSI1.
Data are sent from ZC706 in groups of gour 16bit words.
The SSI1 RX timeout interrupt is enabled.
The code then waits for the interrupt to fire.
"void SSI1IntHandler(void)" contains the interrupt routine
Once the interrupt is fired, the data from slave RX FIFO is read,
then checked and assembled to a 32 bit word
The 32bit words are stored in an array ( int32_t g_ulDataRx2[NUM_SSI_DATA])
for further processing and display.
The UART0 is used to send information to host at 115200 baud and 8-n-1
mode.

This example uses the following peripherals and I/O signals on EK-TM4C123GXL.

  • SSI1 peripheral
  • GPIO Port D peripheral (for SSI1 pins)
  • SSI1CLK - PD0 - Clock
  • SSI1Fss - PD1 - Enable
  • SSI1Rx - PD2 - MOSI
  • SSI1Tx - PD3 - MISO (currently not used)

    The following UART signals are configured only for sending console
    messages back to host. They are not required for operation of the SSI1:

  • UART0 peripheral
  • GPIO Port A peripheral (for UART0 pins)
  • UART0RX - PA0
  • UART0TX - PA1

Interrupts

This code uses the following interrupt handlers:

  • SSI1IntHandler.
